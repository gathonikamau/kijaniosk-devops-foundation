# DevOps Delivery Notes

## Flow

Flow is achieved by using a structured Git workflow:

- Work is developed in small increments using feature branches (`feature/starter-kit-files`)
- Changes are merged into `develop` via Pull Requests
- This ensures continuous progress without blocking the main codebase

## Feedback

Feedback is implemented through:

- Early detection of issues through peer review
- Iterative improvements based on review comments
- Pull Requests (PRs) where changes are reviewed before merging

## Learning

Learning is incorporated by:

- Documenting architectural decisions
- Continuously improving design choices as the system evolves
- Reflecting on feedback received during reviews
